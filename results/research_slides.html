    The Task

    Will this recipe be a Hit or a Miss?

    Each recipe carries a nutrition profile, ~674 editorial binary tags, and free-text directions & ingredients. The target: Hit (rating ≥ 4.0) vs Miss (rating < 4.0).

    Known label limitation

    ≈9% of recipes carry rating 0.000, which conflates “rated terribly” with “never rated”. All of them land in the Miss class — label noise that likely depresses every model’s ceiling. (research.md §5.3)

    Stage 0 · src/data_foundation.py

    Two files, no shared key — and a Cartesian trap

    What didn’t work

    • Positional join: the CSV and JSON are sorted differently — only 2.8% of rows align.
    • Title-only join: ~2,300 duplicated titles (“Chicken Soup” ×N) → a many-to-many Cartesian explosion of corrupted cross-matched rows.

    The composite-key fix

    Fingerprint each recipe with title + rating + calories + protein + fat + sodium (numbers rounded to 3 decimals so both files format identically), de-duplicate each side, then a validated 1:1 inner join.

    → 18,223 clean, 1:1-validated matches.

    Loadepi_r.csv (20,052) + full_format_recipes.json (20,130)
    Mergecomposite key, validate 1:1
    Binarizey = rating ≥ 4.0
    Split80/20 stratified, before any fitting
    Stage 1 · src/preprocessing.py

    Outliers broke the scaler before any model ran

    StandardScaler

    Scraping artifacts (impossible calorie/sodium values) absorb nearly all variance.

    fat std ≈ 0.034

    The feature space collapses — fatal for distance-based and L2 models.

    vs

    RobustScaler

    Centers on the median, scales by the IQR — both robust to outliers.

    fat IQR ≈ 0.962

    Healthy spread restored on the same column.

    The elegant pairing

    SimpleImputer(strategy="median") + RobustScaler: ~21% of nutrition values are missing; median-imputed values land at exactly 0.0 in scaled space — the neutral “no information” position. Every transformer is .fit() on train only — zero leakage.

    §2.5.1 · The Audit

    The bug that almost shipped: six features of pure zeros

    The first pipeline version defined the keyword lists — but never injected them. The extractor was constructed bare, and its schema-stability contract quietly emitted all-zero has_* columns. No crash, plausible shapes, reasonable metrics.

    Symptom

    All six has_* columns identically 0 on both splits, in every model run. An entire interpretive story was written about features that didn’t exist.

    Diagnosis

    One line: X.filter(like="has_").sum() on the train split. The same class of structural check as reading pca.n_components_ (Lesson #5).

    Fix + Guard

    Keywords moved to CULINARY_KEYWORDS; the pipeline uses make_culinary_extractor(); and the build now raises if any engineered binary is all-zero on train.

    After the fix

    RF Advanced accuracy moved 0.6178 → 0.6239, and all six keyword features entered RF’s top-20 importances. Lesson #7: a feature pipeline should fail loud when a feature is degenerate — models happily train around it and report nothing.

    §2.6 · MLOps Architecture

    Seven independent trainers, one JSON contract

    src/ packagedata_foundation → preprocessing → train_utils. One shared pipeline for CLI + notebooks.
    train_*.py × 7each model owns its entry point; writes results/<slug>/metrics.json + predictions + plots.
    Aggregatorevaluate_all_results.py trains nothing — reads every metrics.json, prints the 7-row verdict.

    The “42 Guarantee”

    The literal 42 is bound in exactly one place: src/_constants.py. Every split, PCA, forest, and MLP reads it from there. Within a pinned environment (requirements.txt), a clean checkout reproduces every metrics.json bit-for-bit.

    Honesty note

    The seed guarantee does not survive dependency upgrades — sklearn behaviour changes between releases. Reproducibility = same code + same data + same environment.

    §2.4 · A Correctness Bug Disguised as a Warning

    The 99-point probability swing

    Logistic Regression’s default lbfgs solver never converged on the sparse 687-dim input — even at max_iter = 5000. The smoking gun was one recipe:

    max_iter = 2000

    p(Hit) = 0.9997

    ⚡

    max_iter = 5000

    p(Hit) = 0.0000

    “Lamb Köfte with Tarator Sauce” — same model, same data, different iteration budget.

    Fix & lesson

    Switched to liblinear (coordinate descent) → clean convergence, reproducible per-recipe probabilities. Aggregate metrics looked fine the whole time: stable accuracy does not imply a converged model. (Lesson #2)

    §3.5 · The Most Informative Finding

    RF breaks the ceiling — and flips the feature ranking

    vs Logistic Regression (Advanced)

    Acc 0.6239 (+2.28pp) · F1 0.6753 (+3.49pp)

    Why the inversion?

    L2 penalises coefficient magnitude → collinear features split the credit and shrink together. Trees pick the single best splitter per node — collinearity stops being a penalty, and continuous features offer far more split points than 0/1 tags.

    Feature utility is a property of the (feature × model class) pair. (Lesson #3)

    §3.6 · One Architecture, Two Fates

    The same network, 4 pp apart — only the stopping rule differs

    Run 1 — no early stopping

    • 53 epochs, training loss → 0.0241 (memorised)
    • Test Acc 0.5728 · F1 0.5936
    • Far below the linear baseline (0.6011)
    • Reproducible: train_mlp.py --no-early-stopping
    →

    Run 2 — early stopping

    • 18 epochs, weights restored to epoch 7 (best val acc 0.6209)
    • Test Acc 0.6126 · F1 0.6632
    • Clears LR: +1.15 pp Acc · +2.27 pp F1
    • Gain over Run 1: +3.98 pp Acc · +6.96 pp F1

    Lesson #6

    ~100k parameters on 14.6k samples is a liability without regularisation. Validation-based early stopping (validation_fraction=0.15, patience 10) is essentially free — and here it flips the verdict from “worse than linear” to “beats linear”.

    §4 Lesson #5 · The Diagnostic-Driven Fix

    PCA collapsed 687 columns into… one

    PCA(0.90) + KNN scored barely above chance. The instinct says “KNN can’t handle sparse binary spaces” — true, but not what actually broke. Reading pca.n_components_ revealed it: residual outlier variance in 4 nutrition columns ate the whole 90% budget → one retained component.

    The fix: drop nutrition before PCA

    A ColumnTransformer drops calories/protein/fat/sodium pre-projection, forcing PCA to evaluate the tags + culinary features on their own variance budget.

    §4 · What Actually Mattered

    Seven lessons (click to expand)

    1 Outliers beat model choice

    A handful of scraping artifacts crushed the standard-scaled space (fat std 0.034). No hyperparameter tuning could fix it — only RobustScaler + median imputation. Diagnose input pathology before tuning models.

    2 Convergence is correctness, not performance

    lbfgs’ warning looked cosmetic while per-recipe probabilities swung 99 points between runs. Stable aggregate metrics ≠ a converged model.

    3 Feature utility is model-dependent

    The engineered numerics sat near the bottom of LR’s |coef| ranking (597, 642 / 687) and at the very top of RF’s importances. Conclusions from one model class can be exactly inverted by another.

    4 F1 alone is unsafe — and select thresholds on validation

    The F1-maximising threshold labels 90.5% of recipes “Hit” (balance 19.4). Inspect balance and the predicted-class distribution alongside F1 — and never select the threshold on the test set.

    5 Read structural diagnostics before theorising

    “KNN fails in sparse spaces” was true but secondary — the real killer was PCA retaining one component, visible in a single line: pca.n_components_.

    6 Capacity is a liability without regularisation

    The unregularised MLP memorised the training set and fell far below the linear baseline; early stopping (free!) flipped it to above-baseline. +3.98 pp Acc / +6.96 pp F1.

    7 Audit that engineered features reach the model

    Six keyword features shipped as all-zero columns — silently. One column-sum check caught it; the pipeline now fails loud on degenerate features. Models will happily train around dead inputs and report nothing.
